# machine-learning (pet projects monorepo)

## О репозитории
Это мой монорепозиторий с пет-проектами, которые я пишу по мере изучения Machine Learning.
Идея простая: один репозиторий, много небольших самостоятельных проектов, каждый в своей папке,
со своим окружением/зависимостями и собственным `README`.

Зачем монорепо:
- удобнее хранить и навигировать по всем учебным проектам в одном месте;
- проще переиспользовать наработки и заметки между проектами;
- единая история прогресса (коммиты, структура, стиль).

## Структура
- каждый проект живет в отдельной директории в корне репозитория;
- внутри проекта лежит свой `README.md` с постановкой задачи, данными, запуском и выводами;
- зависимости и окружение изолированы на уровне проекта (например, через Poetry).

## Проекты

### 1) `movie-fake-rating-data-analysis-project`
Исследование гипотезы о том, что сайт Fandango завышает рейтинги фильмов по сравнению с другими сайтами
и собственной пользовательской оценкой.

- Код и ноутбуки: `movie-fake-rating-data-analysis-project/`
- Описание, выводы и как запустить: `movie-fake-rating-data-analysis-project/README.md`

## Как добавлять новые проекты
Рекомендуемая минимальная структура для нового проекта:
- `my-new-project/README.md` (цель, данные, шаги, выводы, запуск)
- `my-new-project/pyproject.toml` (если проект на Python/Poetry)
- `my-new-project/data/` (если есть датасеты)
- `my-new-project/analysis/` (ноутбуки/эксперименты)
- `my-new-project/src/` или `my-new-project/scripts/` (воспроизводимый код)
